# MES 真实业务流程设计

## 1. 工单主流程

1. ERP 或 APS 下发生产计划。
2. MES 生成工单，绑定产品、BOM、工艺路线、产线、交期和优先级。
3. 工单发布到产线后，工位端只显示当前可执行工单。
4. 操作员按站点扫描条码、报工、采集工艺参数。
5. 质量关口对关键工序执行放行、复判或返修。
6. 工单完工后自动汇总产量、良率、工时、设备稼动。
7. 完工数据回写 ERP/WMS，形成入库和交付闭环。

## 2. 条码流程

1. 系统根据产品类别与工单选择条码规则。
2. 签发条码后进入打印队列。
3. 生产过程在来料、工位、箱码、栈板等节点进行扫码绑定。
4. 条码与工单、设备、人员、时间、工艺参数关联。
5. 包装时建立单件 SN 与箱码、箱码与托盘码的层级关系。
6. 出货或售后场景中，可按任意码值反向追溯。

## 3. 履历追溯流程

1. 用户输入 SN、LOT、箱码或工单号。
2. 系统定位 `trace_record`。
3. 读取关联条码、工单、产品、设备、工位、异常与审批记录。
4. 时间线按站点顺序展示关键事件与结果。
5. 若存在异常或让步接收，追溯页面展示审批状态与责任人。

## 4. 审批闭环

典型审批场景：

- 工单数量变更
- 设备保养延期
- 来料让步接收
- 工艺参数临时放宽
- 追溯补录申请

处理链路：

1. 现场或业务人员发起申请。
2. MES 根据工厂、部门、申请类型路由审批人。
3. 审批结果回写业务单据与审计日志。
4. 如涉及锁站、放行、让步接收，自动解除或保留流程限制。

## 5. 正式落地建议

优先顺序建议：

1. 用户、角色、权限和审计日志
2. 工厂、产线、工位、设备主数据
3. 产品、工艺路线、工单主流程
4. 条码签发与站点扫描
5. 履历追溯与异常审批
6. ERP、WMS、设备采集接口
